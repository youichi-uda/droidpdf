# コードレビュー

変更されたファイルをレビューし、以下の観点でフィードバックを出してください。

## レビュー観点

1. **PDF仕様準拠**: ISO 32000-1 (PDF 1.7) の仕様に沿っているか
2. **APIデザイン**: iText7の概念・構造と整合しているか（名前は独自でよい）
3. **メモリ効率**: ページ単位の遅延読み込みの方針に沿っているか。不要なメモリ確保がないか
4. **スレッドセーフティ**: 単一スレッド前提の設計に反していないか（不要な同期処理が入っていないか）
5. **エラーハンドリング**: Lenient方針に沿っているか（仕様外データはスキップ/ワーニング、クラッシュしない）
6. **Kotlin慣用表現**: Kotlinらしい書き方になっているか（Java直訳になっていないか）
7. **セキュリティ**: 入力バリデーション、バッファオーバーフロー等の脆弱性がないか

## 出力フォーマット

ファイルごとに以下の形式で出力:

```
### [ファイルパス]

- [Critical/Warning/Info] L{行番号}: {指摘内容}
  修正案: {具体的な修正コード}
```

Critical が1件でもあれば冒頭に警告を出すこと。
