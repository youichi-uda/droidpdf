# TODO消化チェック

コードベース内のTODOコメントと仕様書のTODOを一覧化し、状況を整理してください。

## チェック対象

1. **コード内TODO**: ソースコード内の `TODO`, `FIXME`, `HACK`, `XXX` コメントを検索
2. **仕様書TODO**: `android_pdf_library_spec.md` の「開発中に決めること」セクション

## 出力フォーマット

```
## コード内TODO ({件数}件)

### Critical (FIXME)
- {ファイルパス}:L{行番号}: {内容}

### 通常 (TODO)
- {ファイルパス}:L{行番号}: {内容}

### 技術的負債 (HACK/XXX)
- {ファイルパス}:L{行番号}: {内容}

## 仕様書TODO
- [ ] {項目}: {現在の状況}

## 推奨アクション
- {優先度の高いものから順に、次に対応すべきTODOを提案}
```
